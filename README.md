# Project Name - Designo Website
This is a solution to the [Designo agency website challenge on Frontend Mentor](https://www.frontendmentor.io/challenges/designo-multipage-website-G48K6rfUT). 

# The challenge
Users should be able to:
- View the optimal layout for each page depending on their device's screen size
- See hover states for all interactive elements throughout the site
- Receive an error message when the contact form is submitted if:
  - The `Name`, `Email Address` or `Your Message` fields are empty should show "Can't be empty"
  - The `Email Address` is not formatted correctly should show "Please use a valid email address"

# Building the Project
We followed a collaborative workflow using GitHub and Figma to bring this project to life. Here's a summary of the steps our team took to build the project effectively:
- Initialized a GitHub repository to manage version control and enable collaboration. We worked on separate branches for different sections (e.g., Home page, Design pages, Header/Footer), which we later merged via pull requests.
- Used Vite with React to set up a fast and modern development environment. File structure was carefully organized into reusable components, page folders, and assets for better maintainability.
- Imported and analyzed the Figma design, paying close attention to dimensions, spacing, colors, and typography to match the visuals pixel-perfectly.
- Created reusable components, which helped reduce repetition and made the UI consistent across pages.
- Implemented responsive and ensured that all elements adjusted properly for tablet and mobile breakpoints.
- Focused on hover states, alignment, and image placement to reflect the interaction design in the original Figma file.
- Tested thoroughly on different screen sizes to ensure responsiveness and visual accuracy.
